Sort Analysis


I hypothesized that the most significant factor that impacts runtime was the number of changes in the list because of the following evidences:

My hypothesis can be explained by using the results of bubble sort and insertion sort and how they function.

- A bubble sort algorithm runs again and again comparing and swapping each element that is in the wrong order until the whole list is sorted, therefore resulting in the greatest number of changes in the array.
- An Insertion sort algorithm pushes the element to the front until there is a number less than the element for each pass, resulting in a lesser number of changes in the list.
- The graph shows that the runtime of bubble sort was longer than insertion sort in all three types of files, and this was because insertion sort had less number of changes than the bubble sort due to differences in processes as explained above.

It can also be explained by the overall runtime difference among the three files

- The graph shows the runtime for sorting ordered files to be the lowest because ordered list doesnâ€™t require any changes in the list for bubble sort and insertion sort.
- The longest runtime came from sorting reverse ordered files because reverse ordered files maximizes the number of changes.
- The runtime for random files was in between ordered and reverse-ordered because there could have been some portions where the words were already in order, causing less amount of change in the list.

It can also be explained by the results of the selection sort

The time taken for selection sort to sort all three files were about the same because the process was identical in all the files due to how selection sort works. The algorithm selects the element with the lowest value and places it in the front, then the second lowest value to the next. This process is the same regardless of how the list is ordered, resulting in similar runtimes as shown in the graph. While bubble sort and Insertion sort varied in runtime when running ordered and reverse-ordered due to differences in number of changes in the list, selection sort remained constant in all three files because the number of changes in the list was identical.
